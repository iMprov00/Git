
***[[git init]]*** - инициализация репозитория (используется когда нужно инициализировать новый репозиторий в Git системе)

***[[git add]]*** - добавить в индекс (промежуточное состояние) файл для коммита (перед коммитом нужно добавить файлы которые система Git должна отследить)

***[[git commit]] -m "name_file*** - создать снимок (коммит) 

***[[git status]]*** - проверить состояние репозитория 
*Пример вывода:*
	[[git status]] -s 
	M README.md
	A test.txt
	?? config.json
*Где:*
	М - модификация
	А - добавлен в индекс
	?? - неотслеживается

***[[git diff]]*** - посмотреть что изменилось в репозитории с последнего коммита

***[[git log]]*** - посмотреть логи коммитов

***[[git config]]*** - установить конфигурации репозитория
*Например:*
	***[[git config]] --global user.name*** - установить глобальное имя пользователя

***[[git restore]]*** - восстановить файл до состояния нового коммита

***[[git checkout]]*** - откатить изменения до последнего коммита

***[[git clean]] ==-n***== - проверить что будет удалено (удаляет файлы и директории)

***[[.gitkeep]]*** - такой файл указывается в пустой директории чтобы система гит могла инициализировать директорию

***[[git branch]]*** - создать новую ветку

***[[git switch]] -c*** - создать новую ветку и сразу переключиться на неё (либо без флага **-с** если требуется просто переключиться)

***[[git stash]]*** - временно зафиксировать изменения 

***[[git merge]]*** - слить ветку
